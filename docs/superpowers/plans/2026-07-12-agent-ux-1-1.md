# GoreGraph 1.1 Agent UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Raise GoreGraph's agent usefulness from roughly 7.5–8/10 toward 10/10 by turning existing evidence, trace, diagnostics, coverage, and diff data into compact task-oriented entry points.

**Architecture:** Keep Schema 2 facts and existing analyzers as the source of truth. Add small bounded projections in the agent service, Query, MCP, and Markdown reports; extend the existing workspace diff and diagnostic grouping instead of creating competing models. Every result must retain stable evidence IDs, explicit uncertainty, coverage, and artifact-level freshness.

**Tech Stack:** Go standard library, existing `internal/scan`, `internal/agent`, Query and MCP layers, generated JSON/Markdown, `go test`, `go vet`.

## Global Constraints

- Preserve Schema 2 meanings; new fields and task names are additive.
- Do not add runtime project execution, network access during scans, telemetry, or external dependencies.
- Agent results must default to bounded output and must not require loading the 17 MB dashboard or the complete workspace output.
- JSON and Markdown must be deterministic apart from explicitly documented generation metadata.
- Missing analysis is coverage or uncertainty, never proof that behavior is absent.
- Reuse `evidence_ids`, confidence, resolution, severity, and coverage values already emitted by GoreGraph.
- Each task must include Query, MCP, documentation, and Doctor/integrity coverage where applicable.
